# 前后端数据格式对接说明

## 问题描述

前端页面显示空白或数据无法加载，原因是前后端数据格式不一致。

## 数据格式对比

### 后端返回格式（EmailMessage.to_dict）

```python
{
    'message_id': 'xxx',        # ✓ 字段名是 message_id
    'sender_id': 'xxx',
    'recipient_id': 'xxx',
    'subject': 'xxx',
    'body': 'xxx',
    'timestamp': 1234567890,    # ✓ Unix 时间戳（秒）
    'read': False,               # ✓ 有 read 字段
    'attachments': []
}
```

### 前端期望格式（修复前）

```javascript
{
    id: 1,                      # ❌ 错误：后端没有 id 字段
    message_id: 'xxx',
    sender_id: 'xxx',
    recipient_id: 'xxx',
    subject: 'xxx',
    body: 'xxx',
    direction: 'received',       # ❌ 错误：后端没有 direction 字段
    timestamp: 1234567890
}
```

## 修复内容

### 1. Inbox.jsx - 修复字段名

**修复前：**
```javascript
key={email.id}                    // ❌ 使用不存在的 id
selectedEmail?.id === email.id     // ❌
```

**修复后：**
```javascript
key={email.message_id}                         // ✓ 使用 message_id
selectedEmail?.message_id === email.message_id // ✓
```

### 2. 移除测试数据覆盖逻辑

**修复前：**
```javascript
useEffect(() => {
  if (inbox.length === 0) {
    const testEmails = [...];
    useP2PStore.setState({ inbox: testEmails }); // ❌ 会覆盖后端数据
  }
}, [inbox.length]);
```

**修复后：**
```javascript
const loadInbox = async () => {
  try {
    const response = await fetch('http://localhost:8102/api/inbox');
    const result = await response.json();
    if (result.success) {
      setInbox(result.data || []); // ✓ 优先使用后端数据
    }
  } catch (error) {
    console.error('加载收件箱失败:', error);
    // 只在后端不可用时使用测试数据
    const testEmails = [...];
    setInbox(testEmails);
  }
};
```

### 3. 统一测试数据格式

**修复后（与后端一致）：**
```javascript
{
  message_id: 'test-msg-001',     // ✓ 使用 message_id
  sender_id: '707c12e8...',
  recipient_id: '707c12e8...',
  subject: '欢迎使用P2P SecureMail',
  body: '...',
  timestamp: Date.now() / 1000,   // ✓ Unix 时间戳
  read: false                    // ✓ 添加 read 字段
}
```

## API 端点对应关系

| 前端调用 | 后端接口 | 返回格式 |
|---------|---------|---------|
| `GET /api/health` | ✓ | `{success: true, status: 'running', node_id: 'xxx', timestamp: xxx}` |
| `GET /api/inbox` | ✓ | `{success: true, data: [EmailMessage]}` |
| `GET /api/sent` | ✓ | `{success: true, data: [EmailMessage]}` |
| `GET /api/contacts` | ✓ | `{success: true, data: [{id, node_id, name, email, created_at}]}` |
| `POST /api/send-email` | ✓ | `{success: true, message_id: 'xxx'}` |
| `POST /api/stop` | ✓ | `{success: true, message: '...'}` |

## 字段映射表

| 后端字段 | 前端字段 | 说明 |
|---------|---------|------|
| `message_id` | `message_id` | ✓ 已统一 |
| `sender_id` | `sender_id` | ✓ 已统一 |
| `recipient_id` | `recipient_id` | ✓ 已统一 |
| `subject` | `subject` | ✓ 已统一 |
| `body` | `body` | ✓ 已统一 |
| `timestamp` | `timestamp` | ✓ 已统一（Unix时间戳） |
| `read` | - | 后端有，前端可选使用 |
| `attachments` | - | 后端有，前端暂未使用 |

## 常见问题

### Q: 为什么页面显示空白？
A: 检查浏览器控制台是否有错误。常见原因：
1. 后端未启动（检查 http://localhost:8102/api/health）
2. 字段名不匹配（如使用 `id` 而不是 `message_id`）
3. 数据格式错误（如 `timestamp` 类型不正确）

### Q: 测试数据和后端数据冲突怎么办？
A: 现在已修复，前端优先使用后端数据。只有当后端不可用时才会使用测试数据。

### Q: 如何清除测试数据？
A: 删除 `p2p-mail-app/src/store/p2p.js` 中的 `initData` 函数，或修改 Inbox.jsx/Sent.jsx 移除 fallback 测试数据。

- ✅ 统一了时间戳格式（Unix秒）
- ✅ API 端点完全对应
