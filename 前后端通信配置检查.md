# 前后端通信配置检查

## 修复的问题

### 1. NavLink 导入错误
**错误：** `Uncaught ReferenceError: NavLink is not defined`

**原因：** Layout.jsx 中使用 `NavLink` 但没有从 `react-router-dom` 导入

**修复：**
```javascript
// 修复前
import { Outlet } from 'react-router-dom';

// 修复后
import { NavLink, Outlet } from 'react-router-dom';
```

### 2. Layout 组件参数错误
**错误：** Layout 组件接收 `{children}` 参数但未使用

**修复：**
```javascript
// 修复前
export function Layout({ children }) {

// 修复后
export function Layout() {
```

### 3. postcss.config.js 模块类型警告
**警告：** `MODULE_TYPELESS_PACKAGE_JSON Warning`

**修复：** 在 package.json 中添加 `"type": "module"`
```json
{
  "type": "module",
  "name": "p2p-secure-mail",
  ...
}
```

## 通信流程

### 1. 健康检查流程

```
前端 P2PStatus (每2秒一次)
  ↓
GET http://localhost:8102/api/health
  ↓
后端 P2PAPIHandler.do_GET
  ↓
返回: {success: true, status: 'running', node_id: 'xxx', timestamp: xxx}
  ↓
前端更新状态为 'running'
```

### 2. 收件箱加载流程

```
前端 Inbox
  ↓
useEffect → loadInbox()
  ↓
GET http://localhost:8102/api/inbox
  ↓
后端 p2p_node.mailbox.get_inbox()
  ↓
返回: {success: true, data: [EmailMessage...]}
  ↓
前端 setInbox(result.data)
  ↓
渲染邮件列表
```

### 3. 发送邮件流程

```
前端 Compose → 用户填写表单
  ↓
点击"发送邮件"按钮
  ↓
POST http://localhost:8102/api/send-email
  Body: {recipient_id, subject, body}
  ↓
后端 p2p_node.send_email()
  ↓
返回: {success: true, message_id: 'xxx'}
  ↓
前端显示"发送成功"
```

## 常见问题

### Q: CORS 错误怎么办？

A: 后端已配置 CORS，但如果仍有问题：
1. 确认后端代码中的 CORS 头
2. 检查浏览器控制台的 Network 标签
3. 查看是否有 OPTIONS 预检请求失败

### Q: 连接被拒绝怎么办？

A: 检查以下几点：
1. 后端是否启动（`python p2p_global.py api`）
2. 端口是否正确（8102）
3. 防火墙是否阻止连接
4. 后端日志是否显示"HTTP API服务器启动"

### Q: 数据格式不匹配怎么办？

A: 确认以下字段：
- 后端使用 `message_id` 不是 `id`
- 时间戳是 Unix 秒（不是毫秒）
- 邮件对象包含 `read` 字段

### Q: 状态更新不及时怎么办？

A: 检查以下几点：
1. 是否调用了 `setInbox()` / `setSent()` 等
2. Zustand store 是否正确配置
3. 组件是否订阅了正确的状态
