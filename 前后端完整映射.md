## 🌐 前端路由 → API端点映射

| 前端路由 | 页面组件 | API端点 | HTTP方法 | 说明 |
|---------|---------|---------|----------|------|
| `/` | App.jsx | - | - | 主页面，初始化store |
| `/inbox` | Inbox.jsx | `/api/inbox` | GET | 获取收件箱邮件列表 |
| `/sent` | Sent.jsx | `/api/sent` | GET | 获取已发送邮件列表 |
| `/compose` | Compose.jsx | `/api/send-email` | POST | 发送新邮件 |
| `/contacts` | Contacts.jsx | `/api/contacts` | GET/POST | 获取/添加联系人 |
| `/settings` | Settings.jsx | `/api/stop` | POST | 停止P2P节点 |
| `/test` | TestPage.jsx | `/api/health` | GET | 健康检查 |

## 🔄 完整数据流示例

### 发送邮件流程

```
用户操作
  ↓
Compose组件
  ↓
fetch POST /api/send-email
  { recipient_id, subject, body }
  ↓
P2PAPIHandler.do_POST()
  ↓
p2p_node.send_email()
  ↓
1. 创建EmailMessage对象
2. 使用共享密钥加密邮件
3. 通过DHT查找接收者
4. 发送加密邮件到接收者节点
  ↓
接收者节点接收邮件
  ↓
解密并添加到mailbox.inbox
  ↓
保存到文件 (mailbox/<node_id>/inbox.json)
  ↓
返回响应给前端
  { success: true, message_id: "..." }
  ↓
前端显示"发送成功"
```

### 查看收件箱流程

```
用户访问 /inbox
  ↓
Inbox组件加载
  ↓
useEffect 触发 loadInbox()
  ↓
fetch GET /api/inbox
  ↓
P2PAPIHandler.do_GET()
  ↓
p2p_node.mailbox.get_inbox()
  ↓
从 mailbox/<node_id>/inbox.json 读取
  ↓
返回邮件列表
  { success: true, data: [EmailMessage, ...] }
  ↓
useP2PStore更新状态
  ↓
UI重新渲染，显示邮件列表
```

## 🛠️ 错误处理

### 常见错误代码

| HTTP状态码 | 说明 | 前端处理 |
|-----------|------|---------|
| 200 | 成功 | 正常处理数据 |
| 400 | 请求参数错误 | 提示用户填写完整信息 |
| 404 | 路径不存在 | 记录错误日志 |
| 500 | 服务器内部错误 | 提示用户稍后重试 |

### 错误响应示例

```json
{
  "success": false,
  "error": "节点未启动"
}
```

```javascript
try {
  const response = await fetch('http://localhost:8102/api/inbox');
  const result = await response.json();

  if (!result.success) {
    throw new Error(result.error || '请求失败');
  }

  // 处理成功数据
  const emails = result.data;
} catch (error) {
  console.error('加载邮件失败:', error);
  alert('加载失败：' + error.message);
}
```

## 📝 注意事项

1. **CORS配置**: 所有API端点已配置CORS，允许跨域请求
2. **端口占用**: 确保8000(P2P), 8102(API), 5173(前端)端口未被占用
3. **数据持久化**: 邮件存储在 `mailbox/<node_id>/` 目录下
4. **联系人管理**: 联系人存储在SQLite数据库 `mailbox/p2p_mail.db`
5. **加密安全**: 所有邮件通过端到端加密传输
