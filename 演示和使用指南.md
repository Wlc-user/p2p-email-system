## 🔐 安全特性说明

### 1. 端到端加密
- **算法**: X25519 ECDH + ChaCha20-Poly1305
- **原理**: 
  - 发送方生成临时密钥对
  - 使用接收方公钥计算共享密钥
  - 使用共享密钥加密邮件内容
  - 服务器无法解密邮件内容

### 2. 密码保护
- **算法**: PBKDF2 + SHA256
- **参数**: 100000次迭代, 16字节盐
- **存储**: 盐值 + 哈希值(base64编码)

### 3. 登录防爆
- **限流**: 按IP地址追踪
- **规则**: 5次失败后封禁15分钟
- **自动解封**: 封禁时间过后自动解锁

### 4. 发送限流
- **规则1**: 每分钟最多5封邮件
- **规则2**: 每小时最多50封邮件
- **目的**: 防止DOS攻击

## 🎓 技术要点

### 1. 加密技术
```
X25519: 椭圆曲线Diffie-Hellman
- 256位安全级别
- 计算效率高
- 密钥交换安全

ChaCha20-Poly1305: AEAD加密
- 认证加密
- 速度快
- 完整性保护

PBKDF2: 密码派生
- 抗暴力破解
- 可配置迭代次数
- 随机盐值
```

### 2. 安全设计
```
最小权限: 服务器无法查看邮件内容
纵深防御: 多层限流、多重验证
审计日志: 所有关键操作记录
```

### 3. 并发处理
```
线程池: ThreadPoolExecutor
数据库: SQLite WAL模式
异步: 建议(async/await)
```
