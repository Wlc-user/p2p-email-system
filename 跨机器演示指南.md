# 跨机器演示指南 - 真正的P2P通信

## 🌍 为什么当前演示不显示"跨区域"？

**当前情况**:
- Alice和Bob都在同一台机器上运行
- IP地址: 127.0.0.1 (本地回环)
- 连接类型: Local Loopback (本地回环)
- 不涉及网络传输

**要演示"跨区域"**:
- 需要两台不同的机器
- 或者一台机器 + 一台云服务器
- 通过公网IP建立连接

#### 步骤2: 云服务器上运行Bob
```bash

cd /root/ant-coding-main
python3 -c "
import sys, os, asyncio

node = P2PEmailNode(seed='bob@example.com', port=8101)
print('Bob节点信息:')

```

#### 步骤3: 本地运行Alice连接到云Bob
```bash
# 使用云服务器的公网IP
# 假设云服务器IP: 123.45.67.89

python -c "
import sys, os, asyncio
sys.path.insert(0, 'ant coding/p2p')
from p2p_global import P2PEmailNode, DHTNode

# 创建Alice
alice = P2PEmailNode(seed='alice@example.com', port=8100)
await alice.start()

# 连接到云服务器上的Bob
bob_pub_key = bytes.fromhex('Bob的公钥')  # 填入Bob的公钥
bob_node_id = 'Bob的节点ID'  # 填入Bob的节点ID
bob_ip = '123.45.67.89'  # 云服务器IP
bob_port = 8101

# 交换密钥
alice.encryption.derive_shared_secret(bob_pub_key, bob_node_id)

# 添加到DHT
alice.dht.add_node(DHTNode(bob_node_id, bob_ip, bob_port))

# 发送邮件
await alice.send_email(
    recipient_id=bob_node_id,
    subject='Hello from Local Alice!',
    body='这封邮件从本地发往云服务器'
)

print('[OK] 跨机器邮件发送成功!')
"
```

## 📊 真实跨区域演示的困难

### 为什么不推荐真实跨机器演示？

1. **网络环境不确定**
   - 面试现场网络可能受限
   - NAT穿透可能失败
   - 防火墙可能阻挡

2. **设备限制**
   - 需要两台设备
   - 需要稳定的网络连接
   - 云服务器需要额外成本

3. **时间成本**
   - 准备时间长
   - 调试困难
   - 演示失败风险高

## 💡 面试话术

### 关于"如何知道是跨区域"的提问

**问题**: "你怎么知道Alice和Bob真的在跨区域？"

**回答**:
> "很好的问题！当前演示中，Alice和Bob都在同一台机器上（本地回环）。
>
> 真实场景下，跨区域通信有几个标志：
>
> 1. **IP地址**: 公网IP不同，可以反向查询到地理位置
>    - 例如: 116.x.x.x 是中国，162.x.x.x 是美国
>
> 2. **网络延迟**: 跨区域会有明显的延迟
>    - 本地: <1ms
>    - 跨区域: 50-200ms
>
> 3. **NAT穿透**: 如果是跨区域，会看到STUN/TURN日志
>    - 'STUN: Discovered public IP'
>    - 'TURN: Relay connection established'
>
> 4. **代码设计**: 系统支持全球7个区域STUN服务器
>    - 美国、欧洲、亚洲、澳大利亚等
>
> 所以，通过IP地址、网络延迟、STUN日志就能判断是否跨区域。"
