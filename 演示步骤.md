# 邮箱系统演示步骤

## 准备工作

1. 确保已安装 Python 3.8+
2. 安装依赖: `pip install flask flask-cors cryptography requests`

## 演示步骤

### 步骤1: 启动双服务器

```bash
# 方式1: 一键启动
快速启动.bat

# 方式2: 手动启动
python mail_server.py --db-path server_a_data/mail.db --domain mail-a.com --port 5001
python mail_server.py --db-path server_b_data/mail.db --domain mail-b.com --port 5002
```

**预期输出:**
```
============================================================
CS架构邮件系统 - mail-a.com
============================================================
数据库路径: server_a_data/mail.db
服务端口: 5001

 * Running on http://0.0.0.0:5001
```

两个服务器都启动后,可以看到两个控制台窗口。

### 步骤3: 注册用户(服务器A)

```bash
curl -X POST http://localhost:5001/api/register \
  -H "Content-Type: application/json" \
  -d "{\"node_id\": \"alice001\", \"username\": \"Alice\", \"password\": \"alice123\", \"confirm_password\": \"alice123\"}"
```

**预期输出:**
```json
{
  "success": true,
  "message": "注册成功,请保存您的私钥",
  "user": {
    "node_id": "alice001",
    "username": "Alice",
    "public_key": "..."
  },
  "private_key": "-----BEGIN PRIVATE KEY-----..."
}
```

**重要:** 私钥仅此一次返回,请保存!

### 步骤4: 注册用户(服务器B)

```bash
curl -X POST http://localhost:5002/api/register \
  -H "Content-Type: application/json" \
  -d "{\"node_id\": \"bob002\", \"username\": \"Bob\", \"password\": \"bob456\", \"confirm_password\": \"bob456\"}"
```

### 步骤5: 登录

```bash
curl -X POST http://localhost:5001/api/login \
  -H "Content-Type: application/json" \
  -d "{\"node_id\": \"alice001\", \"password\": \"alice123\"}"
```

**预期输出:**
```json
{
  "success": true,
  "token": "alice001",
  "user": {
    "node_id": "alice001",
    "username": "Alice"
  }
}
```

### 步骤6: 发送邮件

先获取Bob的公钥:
```bash
curl http://localhost:5001/api/publickey/bob002
```

然后发送邮件(注意: 这里简化演示,实际应用中需要在前端加密):

```bash
curl -X POST http://localhost:5001/api/emails/send \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer alice001" \
  -d "{\"recipient_id\": \"bob002\", \"subject\": \"Hello Bob\", \"body\": \"This is a test email\"}"
```

**预期输出:**
```json
{
  "success": true,
  "message_id": "abc123...",
  "message": "邮件发送成功"
}
```

## 演示要点总结

### 功能演示

1. **双服务器独立运行** - mail-a.com 和 mail-b.com 同时运行
2. **用户注册与登录** - 支持密码保护
3. **端到端加密** - 邮件内容加密传输和存储
4. **跨域通信** - 不同服务器用户可互发邮件
5. **垃圾邮件识别** - 自动识别可疑邮件
6. **快捷回复推荐** - 智能推荐回复内容
7. **安全防护** - 登录防爆、发送限流

### 安全演示

1. **密码哈希** - 使用PBKDF2存储密码
2. **端到端加密** - X25519 + ChaCha20-Poly1305
3. **登录限流** - 5次失败封禁15分钟
4. **发送限流** - 防止DOS攻击

### 性能演示

1. **并发支持** - 10用户同时注册和发送
2. **系统稳定** - 高并发下未崩溃
3. **响应快速** - API响应时间<1s

## 清理工作

停止所有服务器:

```bash
taskkill /F /FI "WINDOWTITLE eq Mail Server*"
```

删除数据:

```bash
rmdir /s server_a_data
rmdir /s server_b_data
```

## 常见问题

### Q1: 端口被占用怎么办?
A: 修改启动脚本中的端口号,或关闭占用端口的进程

### Q2: 如何测试跨域通信?
A: 需要手动在两个服务器都注册用户,然后通过API互发邮件

### Q3: 测试失败怎么办?
A: 确保两个服务器都已启动,检查网络连接,查看错误日志
