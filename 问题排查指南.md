# P2P邮件系统 - 问题排查指南

## 🔴 当前问题：前端看不到后端数据

### 问题现象
- 后端日志显示运行中，但前端页面显示测试数据
- 浏览器控制台显示API调用失败

### 根本原因
后端运行了错误的模式！

**错误模式**: 运行 `python p2p_global.py` (无参数)
- 执行默认的P2P演示程序
- 不启动HTTP API服务器
- 端口8102没有响应

**正确模式**: 运行 `python p2p_global.py api`
- 启动P2P节点 + HTTP API服务器
- 在端口8102提供REST API
- 前端可以正常调用

## 🛠️ 完整重启流程

### 步骤1: 完全停止当前服务

```bash
# 使用停止脚本
停止系统.bat
```

或手动停止：
1. 关闭所有Python CMD窗口
2. 关闭所有Node.js CMD窗口
3. 等待2秒

### 步骤2: 确认端口已释放

```bash
# Windows
netstat -ano | findstr :8102
netstat -ano | findstr :5173
```

如果端口仍被占用，使用：
```bash
# 清理8102端口
for /f "tokens=5" %a in ('netstat -ano ^| findstr :8102') do taskkill /F /PID %a

# 清理5173端口
for /f "tokens=5" %a in ('netstat -ano ^| findstr :5173') do taskkill /F /PID %a
```

### 步骤3: 使用正确的启动脚本

```bash
# 确保使用修复后的脚本
启动系统.bat
```

观察后端CMD窗口，应该看到：

## 📋 常见错误和解决方法

### 错误1: 前端显示测试数据

**原因**: 后端运行在demo模式，未启动API

**解决**: 使用 `python p2p_global.py api` 启动

### 错误2: 浏览器控制台显示 CORS 错误

**原因**: 后端未配置CORS或配置错误

**解决**: 确认使用 `p2p_global.py` (已配置CORS)

检查响应头:
```javascript
fetch('http://localhost:8102/api/health')
  .then(r => {
    console.log('CORS头:', {
      origin: r.headers.get('Access-Control-Allow-Origin'),
      methods: r.headers.get('Access-Control-Allow-Methods')
    });
  });
```

期望输出:
```javascript
CORS头: {
  origin: "*",
  methods: "GET, POST, OPTIONS, PUT, DELETE"
}
```

### 错误3: 端口8102已被占用

**原因**: 旧的Python进程未完全停止

**解决**:
```bash
# 查找占用进程
netstat -ano | findstr :8102

# 强制终止
taskkill /F /PID <进程ID>
```

### 错误4: 发送邮件失败 "没有共享密钥"

**原因**: 未与接收者交换公钥

**说明**: 这是正常行为。P2P加密需要先交换公钥才能派生共享密钥。

**当前版本**: 主要用于演示，真实密钥交换需要额外实现。

### 错误5: 前端页面空白

**原因**:
1. 前端服务未启动
2. 端口5173被占用
3. 前端代码错误

**解决**:
1. 确认 `npm run dev:vite` 正在运行
2. 访问 http://localhost:5173
3. 打开浏览器控制台查看错误

### 错误6: 数据库文件锁定

**原因**: 多个进程同时访问数据库

**解决**: 完全停止所有进程后重新启动
