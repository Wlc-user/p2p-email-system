# 完整启动指南

```bash
# 方式2: 双击批处理
```

```bash
npm run dev
```

## 访问地址

- **前端**: http://localhost:5173
- **服务器A**: http://localhost:5001
- **服务器B**: http://localhost:5002
- **健康检查**: http://localhost:5001/api/health

## 测试流程

### 1. 注册新用户
1. 打开浏览器访问 http://localhost:5173
2. 自动跳转到登录页
3. 点击"没有账户? 点击注册"
4. 输入用户名 (如: Alice)
5. 可选:设置密码 (至少8位)
6. 点击"注册账户"
7. **重要**: 复制并保存显示的私钥!

### 2. 登录
1. 输入Node ID (注册时显示的40位字符)
2. 如果设置了密码,输入密码
3. 点击"登录"
4. 跳转到仪表盘

### 3. 发送邮件
1. 点击侧边栏"写邮件"
2. 输入收件人Node ID (需要先注册另一个账户)
3. 输入主题
4. 输入正文
5. 点击"发送邮件"

### 4. 查看收件箱
1. 点击侧边栏"收件箱"
2. 查看接收的邮件
3. 点击邮件查看详情

### 5. 撤回邮件
1. 点击"已发送"
2. 点击要撤回的邮件
3. 点击"撤回邮件"

## 故障排除

### 问题: 无法连接到服务器
**解决方案**:
1. 检查服务器是否运行
2. 访问 http://localhost:5001/api/health
3. 重启服务器

### 问题: 数据库锁定
**解决方案**:
1. 停止所有Python进程
2. 删除数据库文件
3. 重新启动服务器

### 问题: 前端无法加载
**解决方案**:
1. 检查依赖是否安装
2. 删除node_modules重新安装
3. 清除浏览器缓存

### 问题: 注册失败
**解决方案**:
1. 检查Node ID格式 (40位十六进制)
2. 密码至少8位
3. 用户名不能重复

## 注意事项

1. **私钥安全**: 私钥只在注册时显示一次,务必保存好
2. **Node ID**: 40位十六进制字符,用于唯一标识用户
3. **密码**: 可选设置,增加账户安全性
4. **加密**: 所有邮件都会端到端加密
5. **跨域**: 支持服务器A和B之间的邮件互通
