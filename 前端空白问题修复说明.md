# 前端空白问题修复说明

## 问题描述
前端页面显示空白，没有组件化，数据没有绑定，功能也没有绑定。

## 根本原因

### 1. 路由配置错误

**问题：** Layout 组件使用了 `<Outlet />`，但在 App.jsx 中传递的是 `{children}`

```jsx
// App.jsx (错误)
<Layout>
  <Routes>
    <Route path="/" element={<Dashboard />} />
    ...
  </Routes>
</Layout>
```

**正确做法：** 应该使用嵌套路由

```jsx
// App.jsx (正确)
<Routes>
  <Route element={<Layout />}>
    <Route path="/" element={<Dashboard />} />
    <Route path="/inbox" element={<Inbox />} />
    ...
  </Route>
</Routes>
```

### 2. 组件缺失导航绑定

**问题：** Dashboard 中的按钮没有点击事件

```jsx
// Dashboard.jsx (错误)
<button className="px-4 py-2 bg-white text-primary-600 ...">
  开始使用
</button>
```

**正确做法：** 添加 `navigate` 跳转

```jsx
// Dashboard.jsx (正确)
<button onClick={() => navigate('/inbox')} ...>
  开始使用
</button>
```

## 常见问题

### Q: 为什么页面空白？
A: 检查以下几点：
1. 路由是否正确嵌套（Layout 作为父路由）
2. Layout 是否使用 `<Outlet />` 而不是 `{children}`
3. 组件是否正确导出（`export function Inbox()`）
4. 浏览器控制台是否有错误

### Q: 按钮点击没反应？
A: 检查以下几点：
1. 是否添加了 `onClick` 事件
2. 是否使用了 `useNavigate()` hook
3. 目标路由是否存在

### Q: 数据没有加载？
A: 检查以下几点：
1. 后端是否启动（http://localhost:8102）
2. API 路径是否正确
3. 是否调用了 `loadInbox()` / `loadSent()` 等
4. 浏览器网络标签是否有请求失败

### Q: 为什么显示测试数据？
A: 当后端不可用时，前端会显示测试数据作为 fallback。
- 检查后端是否运行
- 检查 API 地址是否正确
- 查看浏览器控制台的网络请求

## 技术要点

### React Router 嵌套路由
```jsx
// 父组件 (Layout)
<div className="flex">
  <Sidebar />
  <Outlet /> {/* 子路由渲染位置 */}
</div>

// 路由配置
<Route element={<Layout />}>
  <Route path="/" element={<Dashboard />} />
  <Route path="/inbox" element={<Inbox />} />
</Route>
```

### 数据绑定模式
```jsx
// Zustand Store
const { inbox, setInbox } = useP2PStore();

// 加载数据
useEffect(() => {
  loadInbox();
}, []);

const loadInbox = async () => {
  const response = await fetch('/api/inbox');
  const result = await response.json();
  if (result.success) {
    setInbox(result.data);
  }
};
```
