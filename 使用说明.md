## 项目简介

P2P SecureMail 是一个基于点对点（P2P）架构的加密邮件系统，具有以下特点：

### 核心特性

✅ **端到端加密**
- 使用 X25519 (ECDH) 进行密钥交换
- 使用 ChaCha20-Poly1305 加密邮件内容
- 军用级安全性，只有收发双方可以解密

✅ **去中心化**
- 无需邮件服务器
- 基于P2P网络直接通信
- 防止单点故障

✅ **全球部署**
- 支持多区域STUN/TURN服务器
- 自适应NAT穿透
- IPv4/IPv6双栈支持

✅ **离线存储**
- 邮件本地持久化
- SQLite数据库管理联系人
- JSON文件存储邮件

### 技术栈

**前端**:
- React 18
- Vite 5
- Tailwind CSS
- Zustand (状态管理)
- React Router (路由)

**后端**:
- Python 3.x
- asyncio (异步IO)
- HTTP服务器 (标准库)
- SQLite (数据存储)
- cryptography (加密库)

**P2P协议**:
- DHT (Kademlia-like节点发现)
- UDP/TCP/WebSocket多协议
- STUN/TURN NAT穿透

**必需软件**:

## 功能详解

### 📬 收件箱

**功能**:
- 查看所有收到的邮件

3. 使用操作按钮进行回复/转发/删除

- 查看所有已发送的邮件

- 删除已发送邮件

**使用步骤**:
1. 点击侧边栏 "已发送"
2. 点击邮件查看详情
3. 使用删除按钮删除

**功能**:
- 写新邮件

**功能**:
- 添加新联系人
- 搜索联系人
- 查看联系人详情
- 删除联系人

## 常见问题

### Q1: 前端页面空白，无法加载？

**A**: 检查以下几点：
1. 确认后端服务已启动: `python p2p_global.py api`
2. 确认前端服务已启动: `npm run dev:vite`
3. 打开浏览器控制台，查看错误信息
4. 检查端口8102和5173是否被占用

### Q2: 发送邮件失败，提示"没有共享密钥"？

**A**: 这是因为还没有与接收者交换公钥。P2P加密需要：
1. 双方先交换公钥
2. 使用ECDH派生共享密钥
3. 才能进行加密通信

当前版本主要用于演示，真实P2P通信需要实现密钥交换流程。

### Q3: 收件箱显示测试数据，不是真实数据？

**A**: 这是正常行为。当后端没有邮件数据时，前端会显示测试数据用于演示。要使用真实邮件：
1. 先添加联系人
2. 使用写邮件功能发送邮件
3. 邮件会保存在后端的 `mailbox/<node_id>/` 目录

### Q4: 如何获取我的节点ID？

**A**:
1. 进入 "设置" 页面
2. 查看 "服务器状态" 卡片
3. 节点ID会显示在那里

### Q5: 邮件能发送给不在联系人列表的人吗？

**A**: 可以。只要知道对方的40位节点ID，就可以直接发送。但建议：
1. 先添加到联系人列表方便管理
2. 确保已与对方交换公钥

### Q6: 停止服务器后还能看到邮件吗？

**A**: 可以。邮件已保存到本地文件：
- 收件箱: `mailbox/<node_id>/inbox.json`
- 已发送: `mailbox/<node_id>/sent.json`
- 联系人: `mailbox/p2p_mail.db`

重新启动服务器后，这些数据会自动加载。

### Q7: 如何清空所有数据？

**A**: 删除 `mailbox` 目录即可：
```bash
# Windows
rmdir /s /q mailbox

# Linux/Mac
rm -rf mailbox
```

### Q8: 可以在同一台电脑运行多个节点吗？

**A**: 可以，但需要：
1. 使用不同的端口
2. 使用不同的节点ID（或不同的seed）
3. 修改启动脚本中的端口配置

示例：
```python
# 节点1
node1 = P2PEmailNode(seed="alice@example.com", port=8000)

# 节点2
node2 = P2PEmailNode(seed="bob@example.com", port=8001)
```

### Q9: CORS错误怎么解决？

**A**: 当前版本已配置CORS允许所有来源。如果仍有问题：
1. 确认后端使用的是 `p2p_global.py`
2. 检查HTTP响应头是否包含 `Access-Control-Allow-Origin: *`
3. 清除浏览器缓存后重试

### Q10: 如何进行真实P2P通信测试？

**A**:
1. 在两台不同的电脑上启动节点
2. 互相交换节点ID和公钥
3. 使用STUN服务器发现公网IP
4. 通过NAT穿透建立连接

当前版本主要用于单机演示，真实P2P需要网络环境支持。
