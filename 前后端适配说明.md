### 4. 发送邮件

**前端调用位置**: `src/pages/Compose.jsx:20`

**API端点**: `POST http://localhost:8102/api/send-email`

**功能**: 发送P2P加密邮件

**请求示例**:
```javascript
const response = await fetch('http://localhost:8102/api/send-email', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    recipient_id: 'recipient_node_id_here',
    subject: '邮件主题',
    body: '邮件内容'
  })
});
const result = await response.json();
```

**请求体**:
```json
{
  "recipient_id": "707c12e8dd7dc34001e5dbe76aabaec89444440c",
  "subject": "邮件主题",
  "body": "邮件内容"
}
```

**响应示例**:
```json
{
  "success": true,
  "message_id": "xyz789...",
  "message": "邮件发送成功"
}
```

**后端实现**:
1. 接收请求: `P2PAPIHandler.do_POST()`
2. 创建邮件对象: `EmailMessage(...)`
3. 端到端加密: `p2p_node.encryption.encrypt(plaintext, recipient_id)`
4. 通过DHT路由: `p2p_node.dht.get_nodes(recipient_id)`
5. 发送到接收者: `await self.send_raw(...)`
6. 保存到已发送: `p2p_node.mailbox.add_sent(email)`

## 📊 数据流转图

### 邮件发送流程

```
用户填写邮件表单 (Compose.jsx)
    ↓
POST /api/send-email
    ↓
Python API Handler (do_POST)
    ↓
创建 EmailMessage 对象
    ↓
端到端加密 (ChaCha20-Poly1305)
    ↓
DHT查找接收者地址
    ↓
通过P2P网络发送 (UDP/TCP)
    ↓
接收者节点解密并存储
    ↓
更新收件箱和已发送
    ↓
返回成功响应
```

### 邮件接收流程

```
P2P节点监听UDP/TCP
    ↓
接收到 EMAIL_SEND 消息
    ↓
验证发送者身份
    ↓
使用共享密钥解密
    ↓
创建 EmailMessage 对象
    ↓
添加到收件箱 (mailbox/<node_id>/inbox.json)
    ↓
触发邮件处理器回调
    ↓
返回确认消息 (EMAIL_ACK)
```

## 🔐 加密流程

### 密钥交换

```python
# 1. 生成密钥对
private_key, public_key = Identity.generate_keypair()
node_id = Identity.pubkey_to_id(public_key)

# 2. 交换公钥
node1.pub_key → node2
node2.pub_key → node1

# 3. 派生共享密钥 (ECDH)
node1.encryption.derive_shared_secret(node2.pub_key, node2.node_id)
node2.encryption.derive_shared_secret(node1.pub_key, node1.node_id)

# 4. 加密/解密使用共享密钥
ciphertext = node1.encryption.encrypt(plaintext, node2.node_id)
plaintext = node2.encryption.decrypt(ciphertext, node1.node_id)
```

### 邮件加密

```
原始邮件 JSON
    ↓
JSON序列化
    ↓
ChaCha20-Poly1305加密 (使用共享密钥)
    ↓
Base64编码
    ↓
P2P网络传输
```

## 🔧 故障排查

### 前端无法连接后端

**症状**: 前端页面空白或API调用失败

**检查步骤**:
1. 检查后端是否运行: `http://localhost:8102/api/health`
2. 检查端口占用: `netstat -ano | findstr :8102`
3. 检查CORS配置 (已配置允许所有来源)
4. 查看浏览器控制台错误信息

### 邮件发送失败

**症状**: 提示"没有与接收者的共享密钥"

**解决方案**:
1. 确保已与接收者交换公钥
2. 在DHT中注册接收者节点
3. 检查网络连接

### 数据未更新

**症状**: 收件箱或已发送列表为空

**检查步骤**:
1. 检查 `mailbox/<node_id>/` 目录
2. 验证JSON文件格式
3. 检查文件权限
